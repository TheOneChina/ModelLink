Design Proposal v2 · ModelLink 2.0

换骨架：侧栏导航 + 模型链路板

上一版仍在 v1 的单列卡片流里做优化，这一版把骨架换掉：横向窗口（880 × 640）+ 左侧导航 + 四个页面。首页不再是配置表单，而是一块「模型链路板」——把 ModelLink 的本质（Claude 槽位 → 实际模型的映射）直接画出来；配置工作移进「服务商」页的双栏编辑器。

2026-07-13 · v2 替代上一版单列布局 · 定稿后冻结为 docs/design.md

01

概览页 — 模型链路板

打开应用第一眼回答三件事：代理在跑（侧栏底部常驻）、Claude 里的哪个名字对应哪个真实模型（链路板）、改动生效了没有（右上状态 + 主按钮）。链路板同时消化了 v1 里最难解释的「槽位映射」概念——用户不再需要读文档理解为什么 Claude 里显示的是 claude-3-opus-latest。

概览页 · 浅色 · dirty 状态（侧栏「概览」右侧琥珀点 = 有未应用更改的全局提醒）
链路板是首页主角。左列是 Claude 里显示的槽位名（mono、次要色），右列是真实模型 + 服务商标签。映射机制从「需要解释的实现细节」变成「一眼看懂的界面」。
侧栏承担全局状态。底部常驻「代理运行中 + 端口」，任何页面都能看到；「概览」项右侧的琥珀点在有未应用更改时亮起，从别的页面也能感知。
应用动作只在概览页。页头右侧：dirty 时琥珀提示 + 高亮主按钮；已生效时变为绿勾「配置已生效」+ 安静的「重新应用」。编辑一律自动保存。
最近请求给 2 条。让「代理正在工作」有直接证据，点「查看全部」进日志页。
水印移到侧栏底部。两行小字常驻（防篡改护栏逻辑不变），不再占用一整条横栏。
横向窗口更像桌面工具。880 × 640（最小 800 × 560，可调），macOS 隐藏标题栏，侧栏顶部即拖拽区；Windows 标准标题栏。

06

交互要点

应用状态机全局可见。四态 clean / dirty / applying / error；dirty = 当前配置 hash ≠ 上次成功应用的 hash（持久化，重启不丢）。dirty 时：概览页头琥珀提示 + 高亮主按钮、侧栏「概览」项亮琥珀点、服务商页头出现「应用」胶囊——三处同源，一次应用全部熄灭。

自动保存。编辑 400ms 防抖落盘 config.json，无「保存」按钮；「应用到 Claude Desktop」是唯一动作词。

链路板即文档。槽位映射直接可视化；空槽位行内提示去哪里添加。服务商页模型行的 → claude-3-… 与链路板方向语义一致。

引导即首页。零配置时概览页变成预设网格；点预设 → 跳服务商页并聚焦密钥框 → 填完测试 → 回概览应用。整条路径不出现空白页。

8 槽位约束前置。服务商页头常驻 n / 8；达上限时「添加模型」禁用 + Tooltip 解释，不做事后报错。

更新与水印沿用既有方案。UpdateModal（进度/跳过版本/24h 冷却）移植 ClaudeCN 并套本令牌；水印移到侧栏底部两行，MutationObserver 防篡改护栏逻辑不变。

动效原则：页面切换 160ms 淡入 + 6px 位移（framer-motion）；链路板行在应用成功后依次做一次 300ms 的绿色微闪确认（唯一的「表演性」动效，只在 apply 成功时出现一次）；其余交互零装饰动效，尊重 prefers-reduced-motion。
