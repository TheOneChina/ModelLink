Design Proposal v3 · ModelLink 2.0

单屏指挥台：零导航 Bento

与 v2（侧栏导航 + 分页）并列的待选方案。思路相反：不分页面——一块 960 × 620 的仪表盘把链路、服务商、实时请求同时摆在眼前，任何时刻所有信息一览无余。编辑用右侧滑出抽屉（仪表盘不离场），设置收进右上角齿轮弹层。「模型链路板」这个核心想法两版共享，变的是骨架。

01

主界面 — 一屏三块

左侧大块是模型链路（占满两行的主视觉），右上服务商、右下实时请求。顶栏是全局指挥位：左边身份与运行状态，右边永远是应用状态 + 主按钮——不管在做什么，「应用」都在同一个位置。

主界面 · 浅色 · dirty 状态（顶栏琥珀提示 + 高亮主按钮）
零导航。没有侧栏、没有页面切换——链路、服务商、实时请求同屏常驻。对「开一眼确认在工作」的日常使用，这是点击次数最少的形态。
顶栏 = 指挥位。应用状态和主按钮固定在右上角，任何操作过程中都可见可点；左侧「运行中 + 端口」徽章即全局健康度。
链路板占满左侧两行，仍是主视觉（与 v2 共享这个核心想法）；空槽位提示指向右侧服务商块，视线动线一致。
实时请求自带 LIVE 感。2 秒轮询、新行从顶部推入，代理「正在干活」有直接证据；出错行红色状态码一眼可见。
服务商块只做清单：名称、模型数、思考档位、连通点。点击整行打开右侧编辑抽屉（见 §02），仪表盘退后但不消失。
设置收进齿轮弹层（Popover）：外观三态、开机自启、检查更新、关于。低频操作不占常驻空间。水印为底部细条。

05

交互要点

应用状态机常驻顶栏。四态 clean / dirty / applying / error（hash 判定，与 v2 相同）；因为没有分页，一处提示即全局提示——这是单屏方案在状态传达上的天然优势。

编辑不离场。抽屉打开时仪表盘只是变暗；改模型名时链路板同步刷新（改动即时反映在左侧），保存是自动的，抽屉脚注常驻「更改已自动保存」。

链路板行可点。点某一行直接打开对应服务商的抽屉并定位到该模型——链路板不只是展示，也是入口。

实时请求 2 秒轮询，新行顶部推入；tile 标题可点开 Dialog 看全部 100 条。长期空白 + Claude 无法对话时，tile 内显示诊断提示（「请检查是否已应用」）。

8 槽位约束前置。链路板头部 n / 8 常驻；满槽时抽屉内「添加模型」禁用 + Tooltip。

更新与水印沿用既定方案。UpdateModal 移植 ClaudeCN 套本令牌；水印为底部 22px 细条（防篡改护栏不变）。

动效原则：抽屉滑入 200ms（ease-out）+ 遮罩淡入；apply 成功后链路板行依次 300ms 绿色微闪（与 v2 相同的唯一表演性动效）；LIVE 点 2s 呼吸。其余零装饰，尊重 prefers-reduced-motion。
06

v2 vs v3 怎么选

两版共享：模型链路板、应用状态机 + 自动保存、预设引导空状态、色板与字体、水印与更新方案。分歧只在骨架与操作模型：

维度v2 · 侧栏导航（design-proposal.html）v3 · 单屏指挥台（本稿）
骨架880×640 · 侧栏 + 4 个页面960×620 · 零导航 Bento + 抽屉/弹层
信息呈现一次一件事，页页聚焦，界面更「静」全局一览零点击，密度高，界面更「忙」
编辑方式服务商页双栏编辑器（左列表右表单）右侧滑出抽屉，仪表盘不离场
日志独立整页（可慢慢看）常驻 LIVE 小窗（全量开 Dialog）
设置独立页齿轮 Popover
dirty 提示三处同源（页头 + 侧栏点 + 胶囊）顶栏一处即全局
扩展性好——将来加页面（统计、多配置方案）很自然一屏塞满后再加功能只能进弹层
气质系统设置类工具，稳重仪表盘 / 指挥台，直接
适合喜欢层级清晰、逐页操作喜欢一屏掌控、最少点击
